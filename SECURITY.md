# Security Policy

## Supported Versions

We currently support the latest released versions of our projects.  If you
discover a security vulnerability in an unsupported version, we still
appreciate the report but may prioritise fixing it in supported versions.

## Reporting a Vulnerability

If you believe you have found a security vulnerability, **do not create a
public GitHub issue or discuss it on public channels**.  Instead, please
send a detailed report to our security team at
[security@kdtix.com](mailto:security@kdtix.com).  Include steps to
reproduce the issue, the impact and any suggested fixes if available.  We
will acknowledge your report within a reasonable time and keep you
updated as we investigate【452443717429164†L0-L5】.

vulnerability before any disclosure.  We will work with you to ensure
that coordinated disclosure is handled responsibly【696492342447898†L136-L143】.
