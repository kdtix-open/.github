# KDTIX Open‑Source Community Health

This repository contains default community health files for projects under the
`kdtix-open` organization.  When a repository does not include its own
code of conduct, contributing guide, security policy or issue/pull‑request
templates, GitHub will use the defaults defined here【696492342447898†L98-L109】.  Keeping
these files in a dedicated `.github` repository allows us to define and
maintain shared standards from a single place【696492342447898†L126-L129】.

These documents establish clear expectations for how to work with our
open‑source projects: they describe respectful behaviour, how to report
issues, propose changes, and disclose security vulnerabilities.  The files
here were inspired by practices from successful open‑source projects such as
`awesome`, `freeCodeCamp` and others, and follow recommendations from
GitHub’s community health documentation.
